URL Shortener

Build Instructions:

Pre-installation Requirements:

  • Java JDK 8 and above
  • Maven 4.0.0

Setup:

  • Unzip the source code into a directory e.g. URL-Shortener
  • In command prompt, change into source code directory. e.g cd URL-Shortener
  • Compile and build the maven project; e.g mvn clean install

Running:

The above commands will produce a JAR file in the directory URL-Shortener/target which can be simply executed using the command java -jar target/urlshortener-0.0.1-SNAPSHOT.jar 

Usage:

Account Creation:

HTTP Method POST
URI /account
Request Type application/json
Request Body

JSON object with the following parameters:
AccountId (String, mandatory)

Example: { AccountId : 'SalilS' }
Response Type application/json
Response

On successful registration,
you will get JSON response with success = true, system password for the AccountId, and description message of successful registration

Example: 

{
  "password":"7iNxoKLA",
  "success":"true",
  "description":"Account with id SalilS successfully created"
}

On failure, you will get JSON response with success = false and error description message.

Example:

{
  "success":"false",
  "description":"Account with id SalilS already exists"
}

URL Registration:

HTTP Method POST
URI /register
Request Type application/json
Request Headers Authorization header with Basic Authencation token
Request Body

JSON object with the following parameters:
url: (String, mandatory, URL that needs shortening)
redirectType: 301 | 302 (not mandatory, default 302)

Example: { url : 'http://www.123stores.com' }
Response Type application/json
Response

On successful registration,
you will get JSON response with key shortUrl and value as the shortened URL

Example: 

{
  "shortUrl":"baaa7"
}

On failure, you will get JSON response with success = false and error description message.

Example:

{
  "success":"false",
  "description":"Url already registered"
}

Account Statistics:

JSON object with the following parameters:
AccountId (String, mandatory)Example: { AccountId : 'SalilS' }

HTTP Method GET
URI /statistic/{AccountId}
Request Type application/json
Request Headers Authorization header with Basic Authencation token
Response Type application/json
Response

On successful registration,
you will get JSON response with success = true, system password for the AccountId, and description message of successful registration

Example: 

{
  "password":"7iNxoKLA",
  "success":"true",
  "description":"Account with id SalilS successfully created"
}

On failure, you will get JSON response with success = false and error description message.

Example:

{
  "success":"false",
  "description":"Account with id SalilS already exists"
}

 

Redirecting:

Redirecting the client on the configured address with the configured http status.

Example:

On accessing a valid URL; example: http://localhost:8080/baaa7 ; the application will redirect you to registered URL

On accessing an invalid URL, the application will display the application's custom 404 error page
